# 输出契约

## Universal Preamble

Always start with:

1. `六判断`
2. `任务族判断`
3. `mode 判断`

Keep it explicit in `full mode`.

Compress it, but do not drop it, in `light mode`.

## Full Mode

Use this order:

1. `Universal Preamble`
2. `Theme Gate Output`
3. `Strategy Gate Output`
4. `Execution Route`
5. `Adaptive Result Pack`
6. `Evolution Note`

### Minimum Required Fields

#### Theme Gate Output

- chosen theme or brief object name
- why this task should enter now
- what does not enter
- stop condition

#### Strategy Gate Output

- current strategy layer
- target layer if a jump is needed
- core contradiction
- chosen route
- evidence path

#### Execution Route

- chosen skills or tools
- why this route won
- verification target
- human-only checkpoints

#### Adaptive Result Pack

Use the family-specific artifact names from `task-family-map.md`.

#### Evolution Note

- what was gained
- what was wasted
- what should improve next

## Light Mode

Use this order:

1. `Compressed Preamble`
2. `Compressed Theme + Strategy`
3. `Execution Route`
4. `Result`
5. `Short Evolution Note`

### Light-Mode Minimum

Still state:

- whether the task passed the theme gate
- current strategy layer
- chosen route
- verification result
- one evolution sentence

## Human Checkpoints

Only expose checkpoints for:

- missing subjective preference
- authorization
- payment
- irreversible external action
- final publish / release judgment

Do not ask the user to supervise routine recovery.

## Failure Or Stop Output

If the task stops at the theme or strategy gate, say so explicitly.

Use:

- current gate
- why the task stopped
- what would need to change to continue
